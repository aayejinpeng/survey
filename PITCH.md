# Survey Pipeline - 项目简介

## 一句话介绍

> **一个自动化论文调研工具：从 DBLP 批量抓取 → 关键词智能打分 → AI 双重验证提取精华 → 输出开题报告即用素材**

---

## 痛点场景

写开题报告时的典型困境：
```
找论文 → 题录太多不知道哪些有用
↓
筛论文 → 一篇篇看摘要、点开 PDF、手抄笔记
↓
理论文 → 手动整理贡献点、实验数据、研究空白
↓
写报告 → 再回去翻 PDF 找原文引用
↓
耗时：1-2周     痛苦：⭐⭐⭐⭐⭐
```

用 Survey Pipeline：
```
配置关键词 → 一键抓取打分 → Web 快速浏览 → AI 自动提取 → 对照审核
↓
耗时：1-2天     痛苦：⭐
```

---

## 核心功能（4步）

| 步骤 | 功能 | 产出 |
|------|------|------|
| **1. 抓取** | 自动从 DBLP 抓取论文 | 14,600+ 篇题录 |
| **2. 打分** | 按关键词自动排序 | 按相关性排好队 |
| **3. 提取** | AI 提取精华内容 | 结构化 dossier |
| **4. 审核** | 对照 PDF 验证 | 可直接引用的素材 |

---

## AI 双重验证机制

```
┌────────────────────────────────────────────────────────────┐
│                                                         │
│  原始论文                                                 │
│     ↓                                                    │
│  ┌─────────────────┐                                     │
│  │  Claude 分析    │  擅长：长文本理解、结构化输出          │
│  │  (GLM-5.1)      │  输出：研究目的、贡献点、实验数据...  │
│  └─────────────────┘                                     │
│     ↓                                                    │
│  ┌─────────────────┐                                     │
│  │  GPT 审查       │  擅长：验证、纠错、逻辑检查           │
│  │  (GPT-5.4)      │  输出：审查报告 + 修正版本            │
│  └─────────────────┘                                     │
│     ↓                                                    │
│  ┌─────────────────┐                                     │
│  │  人工对照审核   │  对照 PDF，最终把关                   │
│  │  (Web UI)       │  输出：可直接引用的素材               │
│  └─────────────────┘                                     │
│                                                         │
└────────────────────────────────────────────────────────────┘
```

**为什么这样设计？**
- 两个模型独立产生相同幻觉的概率极低
- Claude 的生成能力 + GPT 的审查能力 = 质量保证
- 人工审核在关键节点把关，不盲目信任 AI

## 输出格式：开题报告即用

AI 提取的 dossier JSON 包含：

```json
{
  "research_purpose": "研究目的（中文）",
  "research_significance": "研究意义（中文）",
  "contributions": [
    {"point": "贡献点（中文）", "evidence": "原文引用（英文）"}
  ],
  "key_technique": "关键技术（中文，含设计细节）",
  "key_results": "实验结果（中文，含具体数值）",
  "gap_identified": [
    {"gap": "研究空白（中文）", "evidence": "原文引用", "relevance_to_cute": "与你的研究的关联"}
  ],
  "proposal_evidence": {
    "for_state_of_art": "国内外研究现状论据",
    "for_gap": "研究空白论据",
    "for_feasibility": "可行性论据"
  }
}
```

**直接对应开题报告的章节！**

## 快速开始（30秒）

```bash
# 1. 编辑配置文件
vim configs/venues.yaml    # 要抓哪些会议
vim configs/topic-cpu-ai.yaml  # 你的研究主题关键词

# 2. 一键运行
python3 fetch_dblp.py --config configs/venues.yaml --output-dir data/db/
python3 enrich_papers.py --input-dir data/db/ --output-dir data/enriched/
python3 score_papers.py --input-dir data/enriched/ --topic-config configs/topic-cpu-ai.yaml --output-dir data/topics/cpu-ai/

# 3. 查看结果
head -20 data/topics/cpu-ai/scored.csv
```

## 对比传统方式

| 维度 | 传统方式 | Survey Pipeline |
|------|----------|-----------------|
| 找论文 | 手动搜各个会议网站 | 一键抓取 14,600+ 篇 |
| 筛论文 | 看标题、看摘要、看 PDF | 关键词自动打分排序 |
| 提取信息 | 手抄笔记、复制粘贴 | AI 自动提取结构化数据 |
| 验证准确性 | 凭记忆、手动核对 | AI 双重验证 + 人工审核 |
| 耗时 | 1-2 周 | 1-2 天 |
| 准确性 | 依赖个人能力 | 可验证、可追溯 |
